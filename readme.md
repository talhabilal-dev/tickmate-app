## Overview

TickMate is a full-stack ticket management platform with:

- Role-based authentication
- User and admin dashboards
- Ticket lifecycle management (create, assign, reply, update, complete, soft-delete)
- Async email workflows (verification, password reset)
- AI-powered ticket analysis and vector similarity search
- Audit logging and analytics

### Monorepo Structure

- **tickmate-backend**: TypeScript + Express API server for authentication, ticket lifecycle, admin management, async jobs, and AI-powered features.
- **tickmate-frontend**: Next.js app for user and admin interfaces, connecting to the backend API.

---

## Tech Stack

**Backend:**

- Node.js + Express 5 (ESM)
- TypeScript
- PostgreSQL + Drizzle ORM
- Zod validation
- JWT auth with HttpOnly cookie
- Inngest for async jobs
- Resend for email
- OpenAI (gpt-4.1-mini) for ticket analysis
- OpenAI embeddings (text-embedding-3-small) + Qdrant for similarity search

**Frontend:**

- Next.js, React, TypeScript, PostCSS, Vercel

## Backend Features

- User auth and profile APIs
- Admin-only management APIs
- Ticket lifecycle APIs
- Email verification and password reset workflows
- Similar-ticket search over resolved public tickets

### API Base Paths

- Auth: `/api/auth`
- Tickets: `/api/tickets`
- Admin: `/api/admin`

### Behavioral Rules

- Unverified users cannot log in
- Password reset uses magic-link token
- Ticket replies restricted to moderator/admin
- Ticket edits blocked when completed
- Ticket deletion is soft-delete

## Setup

### 1) Install

```bash
pnpm install
```

### 2) Configure `.env`

Required:

- `DATABASE_URL`, `JWT_SECRET`, `OPENAI_API_KEY`, `QDRANT_URL`, `QDRANT_API_KEY`, `RESEND_API_KEY`, `EMAIL_FROM`
Optional:
- `PORT`, `NODE_ENV`, `APP_URL`, `CORS_ORIGINS`, `COOKIE_DOMAIN`, `INNGEST_EVENT_KEY`, `INNGEST_SIGNING_KEY`, `GEMINI_API_KEY`, `SIMILAR_TICKET_MIN_SCORE`

### 3) Database

```bash
pnpm drizzle:generate
pnpm drizzle:migrate
```

### 4) Run

```bash
pnpm dev
```

Default URL: `http://localhost:3000`

## API Reference (Backend)

### Auth Routes (`/api/auth`)

| Method | Path | Auth | Notes |
|---|---|---|---|
| `POST` | `/register` | No | Body: `name`, `username`, `email`, `password`, `skills?` |
| `POST` | `/verify` | No | Body: `token` |
| `POST` | `/login` | No | Body: `identifier`, `password` |
| `POST` | `/logout` | Yes | Clears auth cookie |
| `POST` | `/forgot-password` | No | Body: `email` |
| `POST` | `/reset-password` | No | Body: `token`, `newPassword` |
| `GET` | `/check-username/:username` | No | Username availability |
| `POST` | `/resend-verification-email` | No | Body: `email` |
| `GET` | `/profile` | Yes | Current user profile |
| `PATCH` | `/profile` | Yes | Body: `name?`, `username?`, `skills?` |
| `PUT` | `/update-password` | Yes | Body: `oldPassword`, `newPassword` |
| `DELETE` | `/profile` | Yes | Delete own account |

### Ticket Routes (`/api/tickets`)

| Method | Path | Auth | Notes |
|---|---|---|---|
| `GET` | `/` | Yes | Get current user's tickets |
| `GET` | `/public-completed` | Yes | Query: `category?`, `skills?` |
| `POST` | `/similar` | Yes | Body: `title`, `description`, `category?`, `limit?` |
| `POST` | `/` | Yes | Create ticket |
| `PUT` | `/status/:id` | Yes | Mark as completed |
| `GET` | `/get-assigned` | Yes | Tickets assigned to current user |
| `PUT` | `/ticket-reply` | Yes | Moderator/admin only |
| `GET` | `/tickets-summary` | Yes | 7-day summary |
| `DELETE` | `/delete-ticket` | Yes | Soft-delete |
| `PUT` | `/edit-ticket` | Yes | Update fields |

### Admin Routes (`/api/admin`)

| Method | Path | Auth | Notes |
|---|---|---|---|
| `POST` | `/login` | No | Admin login |
| `POST` | `/logout` | Admin | Clear admin session |
| `POST` | `/create-user` | Admin | Create user |
| `POST` | `/create-ticket` | Admin | Assign to moderator |
| `GET` | `/users` | Admin | List users |
| `GET` | `/tickets` | Admin | List tickets |
| `GET` | `/ai-usage` | Admin | AI usage stats |
| `GET` | `/audit-logs` | Admin | Audit logs |
| `GET` | `/dashboard` | Admin | Admin stats |
| `PUT` | `/update-user` | Admin | Update user |
| `DELETE` | `/delete-user` | Admin | Delete user |
| `PUT` | `/tickets/toggle-status` | Admin | Change ticket status |
| `DELETE` | `/tickets/delete-ticket` | Admin | Delete ticket |

## Quick API Examples

Verify email:

```http
POST /api/auth/verify
Content-Type: application/json
{
  "token": "<jwt-token>"
}
```

Reset password:

```http
POST /api/auth/reset-password
Content-Type: application/json
{
  "token": "<jwt-token>",
  "newPassword": "NewPass123"
}
```

Change password (logged in):

```http
PUT /api/auth/update-password
Content-Type: application/json
{
  "oldPassword": "OldPass123",
  "newPassword": "NewPass123"
}
```

## Setup

### Backend

1. `cd tickmate-backend`
2. Install dependencies: `pnpm install`
3. Configure `.env` (see backend README for required variables)
4. Setup database:

  ```
  pnpm drizzle:generate
  pnpm drizzle:migrate
  ```

1. Start server: `pnpm dev`

### Frontend

1. `cd tickmate-frontend`
2. Install dependencies: `pnpm install`
3. Set `NEXT_PUBLIC_API_URL` in `.env` to backend API base
4. Start frontend: `pnpm dev`
